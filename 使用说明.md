# Hermes Agent 使用说明

## 当前目录

工作目录：

```powershell
D:\GuojinX\xilu
```

这个目录保存的是 Hermes 的本地工作区，不是 Hermes 主程序源码仓库。

## 快速开始

推荐直接使用包装好的启动脚本：

```powershell
.\start-hermes.ps1
```

它会自动：

- 设置 `HERMES_HOME`
- 读取 `.env`
- 定位本机已安装的 Hermes CLI

## 环境检查

运行：

```powershell
.\set-hermes-env.ps1
```

默认会执行 `hermes doctor`，用于检查：

- Hermes 运行环境
- 配置文件
- Provider 登录状态
- 关键依赖是否正常

## 飞书网关

手动启动飞书网关：

```powershell
python .\start-hermes-gateway-v2.py
```

这个脚本包含了 Windows 下的兼容处理，优先使用它，不建议直接调用原始 gateway 命令。

## 自动守护

安装计划任务：

```powershell
.\install-hermes-service.ps1
```

安装完成后会创建：

```powershell
\Codex\HermesGateway
```

作用：

- 登录时自动启动网关
- 每 5 分钟巡检一次
- 网关异常时自动重启

查看状态：

```powershell
Get-ScheduledTask -TaskPath '\Codex\' -TaskName 'HermesGateway' | Get-ScheduledTaskInfo
```

手动触发：

```powershell
Start-ScheduledTask -TaskPath '\Codex\' -TaskName 'HermesGateway'
```

## 常用命令

启动 Hermes：

```powershell
.\start-hermes.ps1
```

环境检查：

```powershell
.\set-hermes-env.ps1
```

查看日志：

```powershell
Get-Content .\logs\gateway.log -Tail 50
Get-Content .\logs\agent.log -Tail 50
Get-Content .\logs\errors.log -Tail 50
```

实时跟踪网关日志：

```powershell
Get-Content .\logs\gateway.log -Wait -Tail 50
```

## 关键文件

- `.env`：本地密钥和路径配置
- `config.yaml`：Hermes 主配置
- `start-hermes.ps1`：Hermes CLI 启动入口
- `start-hermes-gateway-v2.py`：飞书网关启动入口
- `install-hermes-service.ps1`：计划任务安装脚本
- `bin\ensure-hermes-gateway.ps1`：网关守护脚本
- `obsidian-routes.json`：Obsidian 路由映射
- `cron\jobs.json`：定时任务定义

## 恢复到新机器

1. 克隆仓库
2. 用 `.env.example` 新建 `.env`
3. 填写你自己的密钥和本地路径
4. 登录 Hermes provider
5. 跑一次 `.\set-hermes-env.ps1`
6. 启动飞书网关并验证消息收发
7. 安装计划任务

## 注意事项

1. 不要把 `.env`、`auth.json`、日志、数据库推到公开仓库。
2. 如果修改了密钥，优先同步更新 `.env`，不要把密钥写进脚本和任务提示里。
3. 如果飞书看起来“进程还在但不回消息”，先看 `logs\gateway.log` 和计划任务状态。
